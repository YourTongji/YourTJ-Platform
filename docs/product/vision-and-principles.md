# 产品愿景与原则

> 文档类型：产品基线
>
> 状态：Active
>
> 负责人：Product owner、Community operations、Platform maintainers
>
> 最近核验：2026-07-11，`origin/main@33584db`

YourTJ 是面向同济校园成员的社区平台。论坛是核心公共讨论空间；选课、评课和闭环积分是
共享身份、搜索、治理和基础设施的子域。产品目标不是复制 X/Twitter 的全部功能，而是借鉴
清晰的社交关系、反馈和发现机制，解决校园信息沉淀、可信讨论和互助协作。

## 用户价值

- 学生可以用校园资格进入社区，但以公开 handle 而非校园邮箱参与讨论。
- 课程、教师、选课和课评信息能够被统一检索、比较和长期沉淀。
- 论坛内容有明确板块、上下文和可恢复的治理流程，不退化为无结构的短内容流。
- 关注、通知和私信帮助关系延续，但不能牺牲隐私、安全或公共讨论完整性。
- 管理员通过有理由、有权限、有审计的工具维护社区，而不是直接编辑数据库。

## 产品边界

### 当前定位

- `Current`：论坛、选课、评课和积分共享一个账号与数据库；校园邮箱是资格证明。
- `Current`：积分是贡献产生的闭环虚拟权益，不是可充值、提现或自由转账的货币。
- `Partial`：首页已有社区信息流，但用户关注关系、完整隐私设置和推荐闭环尚未形成。
- `Partial`：当前很多公开读取允许匿名访问，尚未形成明确的全站 public/campus 可见性政策。

### 明确不做

- 不把校园邮箱、学号等资格信息作为公开社交身份。
- 不提供积分充值、提现、法币兑换或无理由自由转账。
- 不声称私信端到端加密；运营访问、备份与举报证据必须如实说明。
- 不在社交图、隐私和治理基础未稳定时优先建设算法推荐、群聊或复杂广告定向。
- 不让管理员通过普通编辑接口冒充用户改写其内容。

## 十二条产品原则

1. 校园资格与公开身份分离；公开面只使用 handle 和经过允许的资料。
2. 公共论坛内容的可见性由板块政策决定，不能因作者关注关系而破坏讨论树。
3. 用户关注、板块/主题订阅、静音和屏蔽是四种不同关系，数据和文案不得混用。
4. PostgreSQL 是业务事实源；搜索、缓存、计数、热榜和 feed 都是可重建投影。
5. 用户内容保存有版本的 canonical source；渲染 HTML 必须走统一安全管线。
6. 所有用户媒体由平台控制的 asset id、状态和引用关系管理，不接受任意外链作为持久事实。
7. 管理权限按 capability 判断；前端按钮隐藏不构成授权。
8. 治理优先可逆，要求理由、审计、当事人通知和可申诉性。
9. 业务生命周期使用显式状态机；不能依靠多个含义不清的布尔字段拼接。
10. 关键副作用必须幂等、可重试、可观测；不能依赖无监督的 fire-and-forget。
11. 通知事件、发送渠道和用户偏好是三个独立模型。
12. 新数据在落库前必须回答用途、可见者、保留期、导出方式和删除方式。

## 参与者

| 参与者 | 主要能力 | 硬边界 |
|---|---|---|
| 匿名访客 | 读取被政策允许公开的页面 | 不创建内容或关系 |
| 校园成员 | 资料、发帖、评论、互动、关注、私信 | 受状态、信任等级、隐私和限流约束 |
| 版主 | 内容审核、有限用户处置、审计读取 | 不能管理同级/更高角色，不能查看任意私信 |
| 管理员 | 平台配置、用户角色、结构与运营管理 | 仍受理由、审计、最近认证和合规红线约束 |
| 系统/服务 | 投影、调度、通知和治理自动化 | 使用独立 actor kind，不伪装成人类账号 |

## 待产品负责人确认的决策

| 决策 | 推荐默认值 | 未决影响 |
|---|---|---|
| 匿名访客可见范围 | 板块声明 `public/campus/staff`，资料默认 campus | 搜索索引、SEO、OSS URL、隐私设置 |
| 关注是否控制论坛可见性 | 只影响 feed、通知和 DM，不影响公共板块串 | 社交图、回复树、缓存与权限 |
| 屏蔽与静音 | 拆成 X-style 双向 block 与单向 mute | 互动、搜索、资料、私信和通知 |
| 密码登录标识 | 校园邮箱；handle 只作公开身份 | 防枚举、找回密码和改名行为 |
| 推广位性质 | 第一阶段仅自营校园/社区信息 | 审核、计费、商业合规与数据分析 |
| 管理员恢复机制 | 线下双人审核的 out-of-band bootstrap | 最终管理员、账号恢复和审计 |

未决项可以做技术调研，但不得通过一个局部 UI 或 migration 默认替产品作出不可逆决定。

## 产品健康指标

指标必须服务于社区质量，不以“停留时长越长越好”为唯一目标：

- 激活：完成资格验证、资料设置和首次有意义行为的比例。
- 贡献：有回复的主题比例、首次回复时长、活跃贡献者留存，而非单纯发帖量。
- 发现：搜索成功率、无结果率、结果点击和从搜索到有效内容的完成率。
- 安全：举报处理时长、重复施害、误判恢复、申诉改判和骚扰私信比例。
- 可靠性：关键 API 成功率、投影漂移、任务失败、通知延迟和恢复演练结果。

埋点遵守数据最小化；查询日志、关系图和私信元数据的保留期需要隐私负责人批准。
